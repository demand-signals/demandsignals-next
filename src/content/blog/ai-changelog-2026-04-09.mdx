## Claude Code
What's new from Anthropic's AI coding assistant

🆕
**New · Third-party setup**
**Google Vertex AI gets its own setup wizard**
No more hunting through documentation to connect Claude Code to Google Cloud. There's now a step-by-step wizard that walks you through authentication, picking your project and region, and getting everything configured properly.

🛠️
**New · Version control**
**Perforce users get read-only file protection**
If you work with Perforce, Claude Code now stops you from accidentally overwriting read-only files. Set `CLAUDE_CODE_PERFORCE_MODE` and it'll tell you to run `p4 edit` first instead of silently making changes you can't save.

📊
**New · Monitoring**
**Watch your background scripts in real-time**
The new Monitor tool lets you see what's happening with scripts running in the background. Think of it like a task manager for your automated workflows.

🔒
**New · Security**
**Better subprocess sandboxing on Linux**
Added PID namespace isolation and script execution limits to keep background processes from going rogue. This one's mainly for teams running Claude Code in shared environments.

🔄
**Improved · Resume feature**
**Picking up where you left off works better**
Fixed a bunch of annoying issues with `/resume` - absolute paths work now, search doesn't reset randomly, and it won't show you stale conversations from other projects.

🔧
**Fixed · Bash permissions**
**Closed several permission bypass loopholes**
Patched multiple ways that commands could sneak past safety checks, including backslash-escaped flags and compound commands. The permission system is now much more reliable at catching risky operations.

🔧
**Fixed · Voice mode**
**No more random spaces in your transcript**
Voice input was adding dozens of extra spaces when you held the push-to-talk button multiple times. Should be much cleaner now.
