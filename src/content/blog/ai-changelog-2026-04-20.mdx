## OpenAI Codex
Major update bringing computer use and expanded capabilities

🛠️
**New · Computer Use**
**Codex now controls your Mac with background computer use**


Codex releases a broad update with marketplace installs from GitHub, git URLs and local sources, better TUI history and memory controls, expanded MCP and realtime APIs, stronger sandbox support

. Multiple agents can work on your Mac in parallel without interfering with your own work. This is basically letting AI actually click and type for you instead of just writing code.

⚡
**New · Plugin Marketplace**
**Install plugins directly from GitHub and git URLs**


Added codex marketplace add and app-server support for installing plugin marketplaces from GitHub, git URLs, local directories, and direct marketplace.json URLs

. No more manual plugin setup — you can now grab community plugins from any git repo and install them instantly.

🔄
**Improved · Terminal Interface**
**Better history search and memory controls in the terminal**


Added TUI prompt history improvements, including Ctrl+R reverse search and local recall for accepted slash commands

. 

Added TUI and app-server controls for memory mode, memory reset/deletion, and memory-extension cleanup

. Finding old commands and managing what Codex remembers just got way smoother.

🤖
**New · MCP Integration**
**Expanded support for Model Context Protocol with parallel calls**


Expanded MCP/plugin support with MCP Apps tool calls, namespaced MCP registration, parallel-call opt-in, and sandbox-state metadata for MCP servers

. This makes Codex work better with external tools and services, especially when you need to call multiple APIs at once.

🔧
**Fixed · Multiple Bug Fixes**
**Stability improvements across sandbox and network handling**
Various stability fixes for Windows sandbox support, remote websocket connections, and MCP cleanup issues. The kind of behind-the-scenes stuff that makes everything work more reliably day-to-day.
