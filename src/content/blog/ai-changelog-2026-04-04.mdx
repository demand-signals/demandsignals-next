# The AI ChangeLog

## TL;DR

Saturday was a quiet day for AI models themselves, but Claude Code (Anthropic's developer tool) dropped a solid update with better AWS setup, cost tracking improvements, and lots of bug fixes. If you're a developer using Claude Code, this is worth updating for the smoother workflow.

## Claude Code (Anthropic)

**What changed:**
- Added an interactive setup wizard for AWS Bedrock — now when you pick "3rd-party platform" during login, it walks you through connecting to AWS step by step
- Pro users can now see detailed cost breakdowns by model and cache usage with the `/cost` command
- Fixed a bunch of annoying bugs, including issues with subagents failing in tmux sessions and problems with extended thinking mode
- Added a "force refresh" setting for enterprise users who need stricter security controls
- Remote Control sessions now use your computer's name as a default prefix (like "myhost-graceful-unicorn" instead of just random words)
- Removed the `/tag` and `/vim` commands to streamline the interface

**Why you should care:** If you're using Claude Code for development work, the AWS setup wizard makes it way easier to connect to Bedrock if you want to use Claude through AWS instead of directly. The cost breakdown feature helps you track exactly how much you're spending on different models. Most of these changes are quality-of-life improvements that make the tool less frustrating to use day-to-day.

This one's mainly for developers — skip if you're just using Claude through the web interface.

## The Bottom Line

Nothing urgent happened yesterday across the major AI platforms. If you're a developer using Claude Code, the update is worth grabbing for the improved AWS setup and bug fixes, but there's no rush. Everyone else can check back Monday for more interesting updates.
