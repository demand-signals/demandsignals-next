## Google Gemini API
What's new from Google's multimodal AI platform

🤖
**New · Tool Control**
**Google's AI can now drive your computer for you**
They added a "Computer Use" capability that lets Gemini directly control your screen, mouse, and keyboard. Think of it like having an AI assistant that can actually click buttons and navigate websites for you, not just tell you what to do.

📊
**New · Music Generation**
**Real-time music creation just landed**
Lyria RealTime is Google's answer to live music generation. You can now get AI-generated music on the fly, which is pretty wild for content creators who need background tracks in real-time.

🔧
**Fixed · Bugs**  
**Various stability improvements and bug fixes**
They cleaned up some issues with the API that were causing problems for developers. Nothing earth-shattering, but your apps should run more smoothly now.

## Claude Code
What's new from Anthropic's AI coding assistant

🛠️
**New · Tool Integration**
**MCP tools can now handle much bigger results without getting cut off**
Previously, if a tool returned a huge result (like a massive database schema), it would get truncated. Now you can override this and pass through up to 500K characters of results, which is huge for database work and complex integrations.

⚠️
**New · Security Controls**
**Added safety switch for shell execution in skills**
New `disableSkillShellExecution` setting lets you turn off inline shell commands in custom skills and plugins. Basically, you can now prevent Claude from running terminal commands if you're working in a sensitive environment.

🔄
**Improved · Performance**
**Faster text processing on Bun runtime**
If you're using Bun (the JavaScript runtime), Claude Code now processes colored terminal output way faster by tapping into Bun's native functions instead of doing it the slow way.

🔧
**Fixed · Bugs**
**Plan mode no longer loses track after container restarts · Deep links support multi-line prompts · Better keyboard shortcuts on various terminals**
Fixed several annoying issues: plan files getting "lost" when containers restart, multi-line prompts in deep links working properly, and keyboard shortcuts behaving correctly across different terminal apps like iTerm2 and Windows Terminal.
