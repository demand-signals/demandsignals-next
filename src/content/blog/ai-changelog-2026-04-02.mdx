# The AI ChangeLog

## TL;DR
Claude Code got a bunch of useful improvements for developers yesterday — you can now pass bigger chunks of data between tools, and they fixed several annoying bugs that were breaking conversation history and keyboard shortcuts.

## Claude Code (Anthropic)

**What changed:**
- You can now pass much larger data (up to 500K characters) between AI tools without it getting cut off — great for things like database schemas
- New setting to turn off automatic shell command execution if you want more control over what runs
- Multi-line prompts now work properly in deep links (those `claude-cli://` URLs)
- Plugins can now include their own executable files and run them directly
- Fixed a bug where conversation history could get lost when resuming sessions
- Fixed keyboard shortcuts (like cmd+delete) that weren't working in various terminal apps
- Fixed issues with plan mode losing track of files after container restarts
- Better guidance when using the `/claude-api` command for building AI agents

**Why you should care:**
If you're using Claude Code for development work, these fixes make it way more reliable — no more losing your conversation history mid-project or dealing with broken keyboard shortcuts. The bigger data limit means AI tools can work with larger files without choking. If you're not coding with Claude Code, this one's not for you.

## The Bottom Line
If you're a developer using Claude Code, definitely update — the conversation history fix alone will save you from some real headaches. Everyone else can skip this one.
